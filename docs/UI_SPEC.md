# UI 화면 명세서 — S-Semi SampleOrderSystem

**기준 이미지**: `docs/*.png`  
**UI 방식**: 콘솔(터미널) 기반, ANSI 컬러 코드 사용

---

## 공통 규칙

### 구분선
```
==========================================================================  ← 주요 구분 (화면 상단, 섹션 분리)
--------------------------------------------------------------------------  ← 보조 구분 (입력 전후, 결과 전후)
```

### 컬러 코딩

| 대상 | 색상 | ANSI |
|------|------|------|
| 메뉴 번호 `[N]` | 하늘색(Cyan) | `\033[96m` |
| 테이블 헤더 | 밝은 파란색(Bright Blue) | `\033[94m` |
| 강조 수치 (총 재고, 대기 건수) | 노란색(Yellow) | `\033[93m` |
| 완료 메시지 | 초록색(Green) | `\033[92m` |
| 안내/경고 메시지 `※` | 노란색(Yellow) | `\033[93m` |
| 일반 텍스트 | 흰색(White) | `\033[97m` |

### 상태 배지 컬러

| 상태 | 색상 | ANSI |
|------|------|------|
| `RESERVED` | 파란색 | `\033[94m` |
| `PRODUCING` | 주황색 | `\033[33m` |
| `CONFIRMED` | 초록색 | `\033[92m` |
| `RELEASE` | 보라색 | `\033[95m` |
| `RUNNING` | 초록색 | `\033[92m` |

### 재고 상태 배지 컬러

| 상태 | 색상 | ANSI |
|------|------|------|
| `여유` | 초록색 | `\033[92m` |
| `부족` | 노란색 | `\033[93m` |
| `고갈` | 빨간색 | `\033[91m` |

### 프로그레스 바
```
[████████░░░░░░░░░░░░]  44%
```
- 전체 20칸, `█` (채움) / `░` (빈칸)
- 여유: 초록, 부족: 노란, 고갈: 빨간 (재고 상태에 따라 색상 연동)

### 입력 프롬프트
```
선택 > _
```
- 모든 화면 하단 공통

```
  ██╔════╝      ██╔════╝      ██╔════╝    ████║
  ███████╗      █████╗        █████╗      ██╔██║
  ╚════██║      ██╔══╝        ██╔══╝      ██ ██║
  ███████║      ███████╗      ███████╗    ██████║
  ╚══════╝      ╚══════╝      ╚══════╝    ╚═════╝

          반도체 시료 생산주문관리 시스템
==========================================================================

시스템 현황   2026-04-16 09:32:15

전체 주문   36건        생산라인   3건 대기         ← 수치: 노란색

--------------------------------------------------------------------------

  [1] 시료 관리                          [2] 시료 주문
  [3] 주문 승인/거절                     [4] 모니터링
  [5] 생산라인 조회                      [6] 출고 처리
  [0] 종료

--------------------------------------------------------------------------

선택 > _
```

### 구현 포인트
- 상단 ASCII Art 로고: `S-Semi` 텍스트를 블록 폰트로 렌더링 (하드코딩 문자열)
- 시스템 현황: 앱 시작 시 JSON에서 집계하여 표시
  - 등록 시료 수 = `samples.json` 항목 수
  - 총 재고 = 전체 시료 `stock` 합산
  - 전체 주문 = `orders.json` 항목 수 (REJECTED 제외)
  - 생산라인 대기 = `PRODUCING` 상태 주문 수
- 메뉴 **2열 배치**: 좌(홀수번호) / 우(짝수번호), 탭 간격 정렬

## 2. 시료 관리 화면 — `[1]`

```
==========================================================================
  [1] 시료 관리
--------------------------------------------------------------------------

  [1] 시료 등록    [2] 시료 목록    [3] 시료 검색    [0] 위로
  선택 > _
```

### 2-1. 시료 등록

```
--------------------------------------------------------------------------
  시료 ID         > S-006
  시료명          > 질화갈륨-2인치
  평균 생산시간   > 45
  수율 (0~1)      > 0.88
  초기 재고       > 100
--------------------------------------------------------------------------
  등록 완료.

  ID              S-006
  시료명          질화갈륨-2인치
  평균 생산시간   45 min/ea
  수율            0.88
  재고            100 ea

선택 > _
```

#### 구현 포인트
- 입력 순서: ID → 이름 → 생산시간 → 수율 → 초기 재고
- 중복 ID 시: `오류: 이미 존재하는 시료 ID입니다. 다시 입력하세요.` 후 재입력
- 수율 범위 검증: 0 < yield ≤ 1.0, 벗어나면 오류 후 재입력
- 등록 즉시 `data/samples.json` 저장

### 2-2. 시료 목록 조회

```
--------------------------------------------------------------------------
  등록 시료 목록  (총 12종)

  ID        시료명                    평균 생산시간    수율     현재 재고
  ──────────────────────────────────────────────────────────────────────
  S-001     실리콘 웨이퍼-8인치       0.5 min/ea      0.92     480 ea
  S-002     GaN 에피택셀-4인치        0.3 min/ea      0.78     220 ea
  S-003     SiC 파워기판-6인치        0.8 min/ea      0.92      30 ea
  S-004     포토레지스트-PR7          0.2 min/ea      0.95     910 ea
  S-005     산화막 웨이퍼-SiO2        0.6 min/ea      0.88       0 ea

  ...외 7종      [N] 다음페이지
  선택 > _
```

#### 구현 포인트
- 한 페이지 5행 표시, `[N]` 입력 시 다음 페이지
- 테이블 헤더: 밝은 파란색
- 재고 0인 항목: 빨간색으로 강조

### 2-3. 시료 이름 검색

```
--------------------------------------------------------------------------
  검색어 > SiC

  검색 결과 (1건)

  ID        시료명                    평균 생산시간    수율     현재 재고
  ──────────────────────────────────────────────────────────────────────
  S-003     SiC 파워기판-6인치        0.8 min/ea      0.92      30 ea

선택 > _
```

#### 구현 포인트
- 부분 일치 검색 (대소문자 무시)
- 결과 없을 시: `검색 결과가 없습니다.`

## 3. 시료 주문 화면 — `[2]`

```
==========================================================================
  [2] 시료 주문
--------------------------------------------------------------------------

  시료 ID    > S-003
  고객명     > 삼성전자 파운드리
  주문 수량  > 200

--------------------------------------------------------------------------
  입력 내용 확인

  시료        SiC 파워기판-6인치   (S-003)
  고객        삼성전자 파운드리
  수량        200 ea

  [Y] 예약 접수    [N] 취소
  선택 > Y
--------------------------------------------------------------------------

  예약 접수 완료.                          ← 초록색

  주문번호     ORD-20260416-0043
  현재 상태    [ RESERVED ]                ← 파란색 배지

  ※ 재고 확인은 [3] 승인 메뉴에서 직접 진행하세요.   ← 노란색

선택 > _
```

### 구현 포인트
- 입력 순서: 시료 ID → 고객명 → 주문 수량
- 존재하지 않는 시료 ID: `오류: 등록되지 않은 시료 ID입니다.` 후 재입력
- 입력 후 확인 화면 출력 → Y/N 선택
- 주문번호 생성: `ORD-YYYYMMDD-NNNN` (orders.json 당일 건수 + 1, 4자리 zero-padding)
- 확정 시 `data/orders.json` 즉시 저장, 상태 `RESERVED`

## 4. 주문 승인/거절 화면 — `[3]`

```
==========================================================================
  [3] 주문 승인/거절
--------------------------------------------------------------------------

  승인 대기 중인 예약 목록  (RESERVED)

  번호    주문번호       고객              시료                    수량      상태
  ──────────────────────────────────────────────────────────────────────────────
  [1]     ORD-0041       LG이노텍          산화막 웨이퍼-SiO2      300 ea    [ RESERVED ]
  [2]     ORD-0042       SK하이닉스        실리콘 웨이퍼-8인치     150 ea    [ RESERVED ]
  [3]     ORD-0043       삼성전자 파운드리  SiC 파워기판-6인치     200 ea    [ RESERVED ]

  승인할 번호 > 3

--------------------------------------------------------------------------
  재고 확인 중...

  시료          SiC 파워기판-6인치      현재 재고   30 ea
  주문 수량     200 ea                  부족분      170 ea  ←  이 수량만큼 생산
```

#### 재고 충분 분기
```
  재고 충분. 즉시 출고 대기로 전환합니다.

  [Y] 승인    [N] 주문 거절
  선택 > Y
--------------------------------------------------------------------------

  승인 완료.                             ← 초록색

  상태 변경    RESERVED  →  [ CONFIRMED ]
  주문번호     ORD-20260416-0043
```

#### 재고 부족 분기
```
  재고 부족. 부족분 170 ea 승인하시겠습니까?  (실생산량 206 ea / 165 min)

  [Y] 승인    [N] 주문 거절
  선택 > Y
--------------------------------------------------------------------------

  승인 완료.                             ← 초록색

  상태 변경    RESERVED  →  [ PRODUCING ]
  주문번호     ORD-20260416-0043
```

#### 거절 분기
```
  [Y] 승인    [N] 주문 거절
  선택 > N
--------------------------------------------------------------------------

  주문 거절 처리 완료.

  상태 변경    RESERVED  →  REJECTED
  주문번호     ORD-20260416-0043
```

### 구현 포인트
- RESERVED 목록 없을 시: `대기 중인 주문이 없습니다.`
- 재고 확인 후 분기:
  - 충분 (재고 ≥ 수량): CONFIRMED 전환 + 재고 차감
  - 부족 (재고 < 수량): 실생산량·총생산시간 계산 표시 → 승인 시 PRODUCING + 생산 큐 등록
- 실생산량 표시: `ceil(부족분 / (수율 × 0.9))`
- 총생산시간 표시: `평균생산시간 × 실생산량` (단위: min)
- 거절 시 재고 변경 없음, REJECTED 저장

## 5. 모니터링 화면 — `[4]`

```
==========================================================================
  [4] 모니터링    2026-04-16 09:32:15
--------------------------------------------------------------------------

  [1] 주문량 확인    [2] 재고량 확인    [0] 뒤로
  선택 > 1

--------------------------------------------------------------------------
  상태별 주문 현황

  [ RESERVED  ]    3건
  [ CONFIRMED ]    8건
  [ PRODUCING ]    3건   ← 생산라인 대기
  [ RELEASE   ]   18건

--------------------------------------------------------------------------
  재고 현황

  시료명                    재고       상태        잔여율
  ──────────────────────────────────────────────────────────────────
  실리콘 웨이퍼-8인치       480 ea     [ 여유 ]    [████████████████░░░░]  80%
  GaN 에피택셀-4인치        220 ea     [ 여유 ]    [████████░░░░░░░░░░░░]  44%
  SiC 파워기판-6인치         30 ea     [ 부족 ]    [█░░░░░░░░░░░░░░░░░░░]   6%
  산화막 웨이퍼-SiO2          0 ea     [ 고갈 ]    [░░░░░░░░░░░░░░░░░░░░]   0%

선택 > _
```

### 구현 포인트
- 서브메뉴 [1] / [2]:
  - `[1] 주문량 확인`: 상태별 주문 건수 (REJECTED 제외)
  - `[2] 재고량 확인`: 시료별 재고 현황 + 프로그레스 바
  - 실제 화면 예시에서는 [1] 선택 시 두 섹션을 함께 표시
- 재고 상태 판별:
  - `고갈`: stock = 0
  - `부족`: stock > 0 이고 활성 주문(PRODUCING + CONFIRMED) 합산 수량 > stock
  - `여유`: 그 외
- 잔여율 계산: `stock / (stock + 활성주문수량) * 100` (또는 최대값 기준 비율)
- PRODUCING 건수 옆 `← 생산라인 대기` 안내 문구 표시

## 6. 생산라인 조회 화면 — `[5]`

```
==========================================================================
  [5] 생산라인 조회   FIFO 방식
--------------------------------------------------------------------------

  생산라인 1개 (단일 라인)    현재 상태: [ RUNNING ]

  ┌─ 현재 처리 중 ────────────────────────────────────────────────────┐
  │  주문번호   ORD-20260416-0038    시료   SiC 파워기판-6인치         │
  │  주문량     80 ea    재고 30 ea  →  부족 50 ea  →  실생산량 61 ea  │
  │             (수율 0.92 / 49 min)                                   │
  │  진행       [██████████████░░░░░░]  72% 완료    예정 09:49         │
  └───────────────────────────────────────────────────────────────────┘

  대기 중인 주문  (FIFO 순)

  순서    주문번호       시료                    주문량     부족분     실생산량    예상 완료
  ────────────────────────────────────────────────────────────────────────────────────
  1       ORD-0040       산화막 웨이퍼-SiO2      150 ea     150 ea     190 ea      11:43
  2       ORD-0043       SiC 파워기판-6인치      200 ea     170 ea     206 ea      14:28
  3       ORD-0044       GaN 에피택셀-4인치      300 ea      80 ea     114 ea      15:02

  부족분 = 주문량 - 재고,  실생산량 = ceil(부족분 / (수율 × 0.9))
  선입선출(FIFO) 방식으로 처리됩니다.

선택 > _
```

### 구현 포인트
- **메뉴 진입 시** 생산 완료 항목 자동 처리:
  - 각 PRODUCING 주문의 `등록 시각 + 총생산시간`이 현재 시각을 지났으면 → CONFIRMED 자동 전환 + 재고 차감
- 현재 처리 중 항목: 박스 형태로 강조, 진행률 프로그레스 바 표시
  - 진행률 = `경과시간 / 총생산시간 × 100`
  - 예상 완료 시각 = `시작시각 + 총생산시간`
- 대기 큐: FIFO 순 번호 표시
- 예상 완료 시각: 현재 처리 중 완료 후 순차 누적 계산
- 대기 주문 없을 시: `대기 중인 주문이 없습니다.`
- 생산라인 전체 유휴 시: 현재 상태 `[ IDLE ]` 표시, 현재 처리 중 박스 생략

```
==========================================================================
  [6] 출고 처리
--------------------------------------------------------------------------

  출고 가능 주문  (CONFIRMED)

--------------------------------------------------------------------------
```

- 번호 선택 → 즉시 RELEASE 전환, 처리일시 기록

- 처리일시는 시스템 현재 시각 (`std::chrono` 또는 `<ctime>`)

## 화면 전환 구조

```
메인
├── [1] 시료 관리
│   ├── [1] 시료 등록
│   ├── [2] 시료 목록 조회  ([N] 다음 페이지)
│   ├── [3] 시료 이름 검색
│   └── [0] 메인으로
├── [2] 시료 주문  (입력 → 확인 → 완료)
├── [3] 주문 승인/거절  (목록 → 선택 → 재고확인 → Y/N → 결과)
```
